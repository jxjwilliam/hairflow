# AI 发型试戴 App — MVP 实施计划

> **版本**: v2.0  
> **关联文档**: [spec.md](./spec.md)  
> **变更**: 技术栈切换为 React Native + Python FastAPI 全栈  
> **预计总工期**: 10 周（技术栈统一后比 v1.0 缩短 2 周）  
> **建议团队**: 2-3 人（全栈 Python/TS，无 Java 依赖）

---

## 1. 前提假设

| 假设 | 说明 |
|------|------|
| 团队规模 | 1 React Native 前端 + 1 Python 全栈后端 + 0.5 QA（可外部协作） |
| 技能栈 | 精通 TypeScript 和 Python；熟悉 React；了解 Docker 基础 |
| 预投入 | 阿里云账号已开通；微信/支付宝开放平台企业资质已有 |
| 工作模式 | 远程协作，2 周一个 sprint，周一站会，周五 demo |
| MVP 目标 | 上线后 1 个月内获得 100 付费用户，验证"用户愿为发型预览付费" |

---

## 2. 团队与分工

| 角色 | 人数 | 核心职责 |
|------|------|----------|
| **React Native 前端** | 1 | 移动端所有页面；Expo 相机/相册集成；微信/支付宝 SDK 集成；平板自适应 |
| **Python 全栈后端** | 1 | FastAPI 全部模块（Auth/User/Payment/Template/Generation）；Celery 异步任务；DB 设计；美图/YouCam API 对接 |
| **QA（兼职）** | 0.5 | 测试用例编写；手动功能测试；支付流程回归测试 |

> 注：DevOps 工作（ECS 配置、CI/CD、OSS/CDN）由 Python 后端兼任，因为 `docker-compose` + GitHub Actions 的运维成本远低于 ACK 集群。

- 部署从 ACK 简化为 Docker Compose 单机
