# ComfyUI Catalog Thumbnail Generation Design

> Date: 2026-07-17
> Status: Approved for implementation planning

## Goal

Replace the 15 text-only `placehold.co` hairstyle thumbnails with synthetic portraits generated by the local Pinokio ComfyUI instance. Each image must closely represent the template's hairstyle so customers can make an informed selection.

The images will be generated locally with `photon_v1.safetensors` from the prompts in `backend/data/templates_comfyui.json`. They will not use photos of real people, external image services, Ollama, or PhotoMaker identity input.

## Architecture

```text
backend/data/templates_comfyui.json
                |
                +--> backend/scripts/generate_thumbnails.py
                |              |
                |              +-- ComfyUI API at 127.0.0.1:8188
                |              |
                |              +--> backend/static/thumbnails/{id}.png
                |              +--> updated thumbnail paths in template JSON
                |
                +--> backend/workflows/txt2img_hairstyle_catalog.json
                              (manual ComfyUI Web UI testing)

FastAPI /static mount --> templates API absolute URLs --> Expo TemplateCard
```

## Components

### Reusable generation script

Create `backend/scripts/generate_thumbnails.py`.

The script will:

1. Read hairstyle settings from `backend/data/templates_comfyui.json`.
2. Select all templates or one template passed with `--id`.
3. Build a text-to-image ComfyUI workflow for each selected template.
4. Submit the workflow, poll its history, download the output, and save it as `backend/static/thumbnails/{id}.png`.
5. Set the generated template's `thumbnail` field to `/static/thumbnails/{id}.png`.
6. Keep the matching entry in `backend/data/templates.json` synchronized when present.
7. Report success and failure per template and exit non-zero if any generation fails.

Supported command-line usage:

```bash
cd backend
python scripts/generate_thumbnails.py
python scripts/generate_thumbnails.py --id m1
python scripts/generate_thumbnails.py --seed 42
python scripts/generate_thumbnails.py --comfyui-url http://127.0.0.1:8188
python scripts/generate_thumbnails.py --dry-run
python scripts/generate_thumbnails.py --force
python scripts/generate_thumbnails.py --fail-fast
```

Existing output files are skipped by default. `--force` regenerates them. A fixed `--seed` makes adjustments reproducible; without one, each template receives a generated seed.

### Manual ComfyUI workflow

Create `backend/workflows/txt2img_hairstyle_catalog.json` as the manual equivalent of the script workflow. It must be directly loadable in ComfyUI Web UI.

Node flow:

```text
CheckpointLoaderSimple
  +--> CLIPTextEncode (positive hairstyle prompt)
  +--> CLIPTextEncode (negative prompt)
  +--> KSampler <-- EmptyLatentImage (512x768)
                 |
                 +--> VAEDecode --> SaveImage
```

Defaults:

- Checkpoint: `photon_v1.safetensors`
- Size: 512x768
- Steps: 25
- CFG: 6.5
- Sampler: `euler_ancestral`
- Scheduler: `normal`
- Denoise: 1.0
- Filename prefix: `catalog_thumb`

The included example prompt will generate the `w1` French-wave style. The workflow metadata and `backend/workflows/README.md` will explain how to replace the positive and negative prompts for other templates.

### Static image serving

Create `backend/static/thumbnails/` and mount `backend/static` in FastAPI at `/static` with `StaticFiles`.

Template JSON stores portable relative paths such as:

```text
/static/thumbnails/m1.png
```

The list and detail template endpoints will convert relative thumbnail paths into absolute URLs using the incoming request base URL. This preserves correct behavior on:

- Web: `http://localhost:8000`
- iOS simulator or Expo Go over LAN
- Android emulator through its configured backend host

External absolute thumbnail URLs remain unchanged. The mobile app requires no change because `TemplateCard` already passes `template.thumbnail` to `expo-image`.

### Generated assets

The 15 generated PNG files should be committed so a fresh checkout has a working visual demo without requiring ComfyUI. The repeatable script remains the source for later visual adjustments.

## Prompt and visual consistency

The script uses each template's existing positive prompt, negative prompt, checkpoint, dimensions, steps, and CFG. It omits PhotoMaker-specific identity input and uses pure text-to-image denoising at `1.0`.

Prompts should consistently request:

- A synthetic Asian male or female model matching the template category
- A front-facing or slight three-quarter portrait
- Studio lighting and a neutral, uncluttered background
- Clearly visible hair with realistic texture
- No hats, sunglasses, text, watermarks, or obstructed hair

The hairstyle description remains the dominant prompt content. Regeneration and prompt tuning can be performed one template at a time with `--id`.

## Error handling

- Check ComfyUI availability before generating; fail with an actionable message if it cannot be reached.
- Reject an unknown `--id` before submitting work.
- Use bounded polling with a timeout rather than waiting indefinitely.
- Log ComfyUI execution errors with the affected template ID.
- Continue after per-template failures unless `--fail-fast` is set.
- Update template JSON only after the corresponding PNG is downloaded successfully.
- Write JSON atomically to avoid corruption if the process is interrupted.
- A missing static file returns a normal HTTP 404 and must not crash the templates API.

## Verification

1. Load `txt2img_hairstyle_catalog.json` in ComfyUI Web UI and generate its default image.
2. Run the script with `--dry-run` and confirm all 15 template configurations.
3. Generate one style with `--id m1`; verify a valid PNG and updated relative path.
4. Run the script without `--force`; verify the existing file is skipped.
5. Run with `--force --seed 42`; verify deterministic regeneration.
6. Request both template API endpoints and verify absolute thumbnail URLs.
7. Fetch a returned thumbnail URL and verify a successful image response.
8. Open `http://localhost:8081/` and confirm the grid shows hairstyle portraits instead of text placeholders.
9. Review all 15 images manually for close hairstyle correspondence before committing them.

## Scope

Included:

- Batch and single-template local thumbnail generation
- A reusable ComfyUI API script
- A manually loadable ComfyUI workflow
- Static serving and portable template URLs
- Generated catalog PNGs

Excluded:

- Changes to PhotoMaker try-on workflows
- Real-person source photos
- CivitAI, LibLib, Hugging Face, OSS, or Ollama integration
- Automatic generation during API requests
- Mobile card redesign
